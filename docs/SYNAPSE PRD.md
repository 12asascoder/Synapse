## The Problem

Today's learning ecosystem is fundamentally broken.

| What Students Do | What Happens |
|---|---|
| Watch courses | Forget concepts |
| Complete modules | Remain unprepared |
| Earn certificates | Lack real opportunities |

### What Platforms Measure Today

| ❌ Wrong Metrics | ✅ Right Metrics |
|---|---|
| Course completion | Actual understanding |
| Watch time | Skill development |
| Attendance | Critical thinking |
| — | Consistency |
| — | Career readiness |

### The Consequences

- Students remain confused about career paths.
- Skills remain unvalidated.
- Learning lacks accountability.
- Employers cannot verify competency.
- Students from non-technical backgrounds feel excluded.
- Interview preparation is fragmented — students juggle scattered resources, have no structured timeline, and receive no personalized feedback on their performance.

> **The world does not need more certificates.**
> **The world needs verified skill development and real interview readiness.**

## Interview Preparation Feature

> A standalone feature accessible from the hub, independent of the daily learning cycle.

### Step 1: Setup Interview

The user provides:

| Input | Format |
|---|---|
| 🏢 Company Name | Text |
| 📅 Interview Date & Time | DateTime picker |
| 📄 Resume | PDF / DOCX upload |
| 📋 Job Description | Text paste or file upload |

### Step 2: AI Analysis

The AI processes the resume and JD to identify:

- **Skill gaps** between resume and job description
- **Likely interview topics** (technical, behavioral, domain-specific)
- **Estimated difficulty level**
- **Recommended focus areas**

### Step 3: Timeline Generation

Based on time remaining until the interview:

| Time Remaining | Plan Type |
|---|---|
| > 7 days | Day-wise plan covering concepts, practice, mock interviews, revision |
| ≤ 7 days | Hour-slot-based intensive plan |

Each slot has specific goals, topics, and deliverables.

### Step 4: Mock Interviews

- AI-conducted at any time
- Simulates the target company's interview style (incorporating company-wise LeetCode problems from sources like [liquidslr/interview-company-wise-problems](https://github.com/liquidslr/interview-company-wise-problems))

| Round Type | Focus |
|---|---|
| 💻 Technical | Coding, system design, domain questions |
| 👔 HR / Behavioral | Soft skills, situational questions |
| 🧩 Problem-Solving | Analytical and reasoning challenges |

- AI asks questions and evaluates responses in real time
- Scores are provided per question and per topic

### Step 5: Performance Tracking

Every mock interview result is stored.

| Metric | Description |
|---|---|
| Score | Per-topic performance |
| Time Taken | Response duration |
| Attempts | Number of attempts per topic |
| Improvement | Score trend over time |
| Weak Areas | Automatically flagged topics |

### Step 6: Post-Interview Analysis

After the user's real interview:

1. User marks the interview as completed
2. AI asks what topics/questions came up
3. User rates their perceived performance
4. AI generates a **Weak Topics Report**

### Step 7: Targeted Re-Interviews

- User can take new mock interviews focused **only on their weak topics**
- AI generates questions specifically targeting those areas
- Progress is tracked until the user achieves mastery in each weak topic

### Step 8: Company DSA Question Bank

For each company, the AI curates a list of commonly asked DSA questions, leveraging datasets like [liquidslr/interview-company-wise-problems](https://github.com/liquidslr/interview-company-wise-problems) for comprehensive company-wise coverage.

| Dimension | Values |
|---|---|
| 📂 Topic | Arrays, Strings, DP, Graphs, Trees, etc. |
| 📊 Difficulty | Easy, Medium, Hard |
| 🔥 Frequency | Most Asked, Frequently Asked, Occasionally Asked |

Users can:
- Filter by topic, difficulty, and frequency
- Attempt questions interactively
- Track solved vs unsolved status
- View solution hints and explanations
